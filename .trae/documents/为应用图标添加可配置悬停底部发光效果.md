## 目标
- 在启动器中的`应用图标`悬停时，图标底部出现柔和发光效果（不挡点击、不影响布局）。
- 发光样式（开关、颜色、不透明度、模糊、厚度、高宽扩散）可在“设置”中配置，并实时应用。

## 设计说明
- 仅作用于启动器的`.app-icon`元素，不改变 Dock 行为（后续可扩展至 Dock）。
- 采用伪元素实现：`.app-icon::after` 绝对定位于底部，使用`radial-gradient(ellipse)`+`filter: blur(...)`生成底部光晕。
- 保持轻量：通过渲染进程的动态样式注入（与现有`updateIconStyles()`同一路径），不引入新库。

## 配置扩展
- 在`config.json`的`icon`下新增`glow`对象（默认值如下），用于发光效果配置：
  - `enabled: true`（是否开启）
  - `color: "#3b82f6"`（光晕主色）
  - `opacity: 0.9`（0~1）
  - `blur: 12`（像素）
  - `height: 10`（像素，光晕厚度）
  - `spread: 40`（像素，左右扩散宽度）
- 文件位置：`configs/config.json`。持久化由`utils/configManager.js`负责，现有`validateConfig()`对`icon`为对象即可，新增字段可兼容。

## 设置页改动
- 页面：`renderer/settings/index.html`、逻辑：`renderer/settings/settings.js`。
- 新增“图标发光”设置分组：
  - 开关：`glow.enabled`（复选框）
  - 颜色：`glow.color`（颜色选择器）
  - 不透明度：`glow.opacity`（0~1 或 0~100% 滑块）
  - 模糊：`glow.blur`（滑块）
  - 厚度：`glow.height`（滑块）
  - 扩散：`glow.spread`（滑块）
- 在表单采集函数中写入/读取：`collectFormData()` 与 `populateForm()`（参考已处理的`icon.*`采集逻辑，`renderer/settings/settings.js:560`附近）。

## 渲染逻辑改动
- 文件：`renderer/renderer.js`
- 在`updateIconStyles()`中注入发光样式（函数定义：`renderer/renderer.js:230`）：
  - `.app-icon { position: relative; }`（确保伪元素定位）
  - 基础伪元素：`.app-icon::after { content:''; position:absolute; left:50%; transform:translateX(-50%); bottom:-2px; width:calc(100% + <spread>px); height:<height>px; pointer-events:none; border-radius:50%/100%; background: radial-gradient(ellipse at center, <rgba(color,opacity)> 0%, transparent 60%); filter: blur(<blur>px); opacity:0; }`
  - 悬停显现：`.app-icon:hover::after { opacity:1; }`
  - 以上参数从`appStore.config.icon.glow`读取；当`glow.enabled=false`时不注入伪元素样式或注入`opacity:0`以禁用。
- 保持与现有样式的协同：不移除`showShadow`、`enableAnimation`等既有能力，发光为叠加效果。

## 实时生效
- 渲染侧已监听：`electronAPI.onConfigUpdate(...)`（`renderer/renderer.js:84`），会调用`updateUIWithConfig()`→`updateIconStyles()`。
- 主进程补充保存后广播：在`main.js:364`的`ipcMain.handle('save-config',...)`中，保存成功后向主窗口发送`config-updated`：`mainWindow.webContents.send('config-updated', configManager.getConfig())`。
- 预加载桥接已提供`onConfigUpdate`（`preload.js:28`）；设置页中误用的`electronAPI.applyConfig(...)`可移除，改为依赖保存后事件广播。

## 默认值与兼容
- 如用户配置缺失`icon.glow`，在`updateIconStyles()`中使用上述默认值防御。
- 不改变现有布局与点击区域；伪元素`pointer-events:none`避免遮挡。
- 同时适配图标缩放：发光宽度与厚度以像素为单位，受外层`transform: scale(...)`一起缩放，效果自然。

## 验证与测试
- 在设置页调节各控件，验证主界面图标悬停光晕是否实时变化。
- 边界值测试：`opacity=0`、`blur=0`、极大`spread`与`height`不破坏布局。
- 兼容验证：与`icon.showShadow`、`icon.size`、`spacing.iconGap`同时启用，确保无样式冲突。
